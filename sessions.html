Sessions & Cookies in PHP

What are Sessions?

Sessions in PHP allow you to store information (variables) to be used across multiple pages. Unlike cookies, which store data on the client side, session data is stored on the server, making it more secure. A session is started using the session_start() function.

1. Starting a Session

To use sessions in PHP, you must call the session_start() function at the beginning of each page that needs access to session data. This function must be called before any HTML or output is sent to the browser.

<?php
session_start(); // Start a session

// Store session data
$_SESSION["user"] = "John Doe";
$_SESSION["email"] = "john@example.com";
?>

Explanation: The $_SESSION superglobal is used to store session data. The session data persists across different pages until the session is destroyed or expired.

2. Accessing Session Data

Once a session is started, you can access the session data on any page by using the $_SESSION superglobal array.

<?php
session_start(); // Start the session

// Access session data
echo "User: " . $_SESSION["user"];
echo "<br>Email: " . $_SESSION["email"];
?>

Explanation: In this example, the session data stored earlier is retrieved and displayed on the page.

3. Modifying Session Data

You can modify the session data by simply reassigning values to the $_SESSION superglobal.

<?php
session_start(); // Start the session

// Modify session data
$_SESSION["user"] = "Jane Smith"; // Change user name
echo "Updated User: " . $_SESSION["user"];
?>

Explanation: In this example, the session variable $_SESSION["user"] is updated with a new value.

4. Destroying a Session

You can destroy all session data using the session_destroy() function. This will delete all the session variables and close the session. However, the session cookie may remain in the browser until it expires.

<?php
session_start(); // Start the session

// Destroy all session data
session_unset(); // Remove all session variables
session_destroy(); // Destroy the session

echo "Session has been destroyed.";
?>

Explanation: The session_unset() function clears all session variables, and session_destroy() ends the session. This ensures that session data is no longer accessible.

What are Cookies?

Cookies are small pieces of data that are stored on the user's computer by the web browser. Cookies can store data such as user preferences, authentication tokens, or other state information. Cookies are sent to the server with every request to the same domain.

5. Setting a Cookie

To set a cookie in PHP, use the setcookie() function. This function requires the name of the cookie, its value, and an expiration time. The cookie will be available to all pages on the domain that set it.

<?php
// Set a cookie for 30 days
setcookie("username", "JohnDoe", time() + 30 * 24 * 60 * 60); // Expire in 30 days
?>

Explanation: The setcookie() function sets a cookie named "username" with the value "JohnDoe". The expiration time is set to 30 days from the current time.

6. Accessing Cookie Data

To access a cookie value, use the $_COOKIE superglobal. Keep in mind that cookies are only available after the page is reloaded (because they are sent with the HTTP request after being set).

<?php
if(isset($_COOKIE["username"])) {
    echo "Welcome back, " . $_COOKIE["username"];
} else {
    echo "Cookie not set.";
}
?>

Explanation: This code checks if the "username" cookie is set and then displays its value. If the cookie is not set, a message is displayed.

7. Modifying a Cookie

To modify a cookie, you simply set a new value for it using setcookie() with the same name and a new expiration time.

<?php
// Modify the cookie value
setcookie("username", "JaneDoe", time() + 30 * 24 * 60 * 60); // New value for 30 days
?>

Explanation: Here, the "username" cookie is updated with a new value ("JaneDoe") and the same expiration time (30 days).

8. Deleting a Cookie

To delete a cookie, set its expiration time to a time in the past. The browser will automatically remove the cookie when it detects the expiration date has passed.

<?php
// Delete the cookie by setting its expiration time to a past date
setcookie("username", "", time() - 3600); // Expire in the past (1 hour ago)
echo "Cookie has been deleted.";
?>

Explanation: By setting the cookie's expiration time to a past date, the browser will remove the cookie on the next page load.
